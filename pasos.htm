Manual Técnico: Creación y Despliegue de Sitio Web

Este documento detalla los pasos esenciales y las decisiones técnicas tomadas durante el proceso de desarrollo de este sitio web personal, desde la conceptualización inicial hasta su publicación.

1. Diseño y Estructura Inicial (HTML)

La fase inicial definió la arquitectura fundamental del sitio, optando por una estructura modular.

  • Se diseñó una estructura base utilizando HTML5.
  • Se implementó un sistema de marcos (frames) para la navegación.
  • Se creó un archivo HTML dedicado al **menú de navegación**.

2. Desarrollo de Contenidos Específicos

Una vez establecida la base, se crearon las páginas individuales que componen el sitio.

  • Página de Hoja de Vida (CV).
  • Página de Notas.
  • Galería de Imágenes.
  • Página de Video: Integración de contenido multimedia (ej., videos de YouTube).
  • Página de Juego.
  • Página de Pasos (este manual)**.
Nota Técnica: La integración de videos de YouTube requiere usar la URL de incrustación (embed URL) y que el video permita la incrustación.

3. Estilización y Mejora Estética (CSS)

Para dotar al sitio de una apariencia moderna, se aplicaron estilos CSS.

  • Se definieron paletas de colores coherentes.
  • Implementación de tipografías legibles.
  • Uso de CSS Grid y Flexbox (o técnicas de posicionamiento) para diseños organizados.
  • Aplicación de sombras, bordes redondeados y efectos de transición/hover**.
Consejo de Diseño: En galerías, object-fit: cover; es clave para que las imágenes mantengan su proporción.

4. Pruebas Locales y Depuración

Antes del despliegue final, se realizaron pruebas exhaustivas para asegurar funcionalidad y compatibilidad.

  • Pruebas de navegación y enlaces.
  • Compatibilidad de navegador (Chrome, Firefox, Edge, etc.).
  • Revisión de contenido y corrección de errores.
  • Validación HTML/CSS.

5. Despliegue y Publicación

El paso final consistió en hacer el sitio accesible al público.

  • Selección de una plataforma de alojamiento web (GitHub Pages).
  • Preparación y subida de los archivos del proyecto al repositorio.
  • Verificación de la URL pública del sitio.
Consideración: GitHub Pages es excelente para proyectos estáticos, ofreciendo un despliegue sencillo y gratuito.
